# 售前工时评估报告 — 示例：物料管理模块新增批次追溯功能

> 本文档为标准输出示例，用于指导 AI 生成格式一致的工时评估报告。

## 一、需求概述

### 1.1 需求背景
客户现有管理系统缺少对物料批次的全链路追溯能力，生产过程中发生质量问题时无法快速定位受影响批次，导致召回范围扩大、处理周期长。客户希望在物料管理模块新增批次追溯功能。

### 1.2 核心痛点
- 质量问题发生时，人工排查批次需要 2-3 天，严重影响响应速度
- 批次与工单、供应商、检验记录分散在多个系统，无统一视图
- 缺少批次谱系图，无法可视化展示上下游关联关系

### 1.3 潜在需求
- 批次预警：当某批次物料被标记问题时，自动通知下游使用方
- 供应商质量报表：基于批次追溯数据生成供应商质量月报
- 与 ERP 系统的批次数据双向同步

### 1.4 风险提示
- 技术风险：批次谱系图涉及树形递归查询，数据量大时性能需关注
- 集成风险：与现有工单模块、检验模块存在数据关联，需协调接口变更
- 数据风险：历史存量数据无批次信息，需确认是否需要数据迁移方案

## 二、功能范围

### 2.1 用户故事拆解

| 用户故事 | 优先级 | 复杂度 |
|---------|--------|--------|
| 作为质量工程师，我希望输入批次号后查看该批次的完整使用记录，从而快速评估质量问题影响范围 | P1 | 中 |
| 作为质量工程师，我希望以图形方式查看批次的上下游关联谱系，从而直观了解物料流向 | P1 | 高 |
| 作为采购员，我希望在新建入库单时必须录入供应商批次号，从而确保批次信息完整 | P1 | 低 |
| 作为系统管理员，我希望能标记问题批次并触发通知，从而及时告知相关人员 | P2 | 中 |

### 2.2 功能清单
- 批次信息录入与维护（入库时强制录入）
- 批次追溯查询（正向：批次→工单→产品；反向：产品→工单→批次）
- 批次谱系可视化（树形图）
- 问题批次标记与通知

## 三、技术方案

### 3.1 涉及模块
- 物料管理模块（主要）
- 工单管理模块（关联）
- 质量检验模块（关联）
- 消息通知模块（关联）

### 3.2 技术栈
- 前端：Vue 3 / G6（谱系图可视化）
- 后端：Java / Spring Boot
- 数据库：PostgreSQL（递归 CTE 查询）
- 消息：现有站内信机制

### 3.3 影响范围
- 涉及对象/字段：`MdMaterial`（新增 batch_no 字段）、新增 `MdMaterialBatch` 实体
- 涉及接口：入库单保存接口（新增批次校验）、新增批次追溯查询接口 x3
- 涉及流程：入库流程（增加批次录入步骤）
